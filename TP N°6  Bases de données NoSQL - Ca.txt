II. Installation et Configuration
1. Vérification de la virtualisation
Avant d'installer Docker, la virtualisation a été activée dans le BIOS (F1 → Config → CPU → Intel Virtualization Technology → Enabled).

2. Installation de WSL2 (Windows Subsystem for Linux)
bash

wsl --set-default-version 2
3. Lancement du conteneur Cassandra
bash
docker run --name mon-cassandra -p 9042:9042 -d cassandra:latest
Statut final : Le conteneur mon-cassandra fonctionne et écoute sur le port 9042.

III. Création de la Base de Données (Keyspace) et des Tables
Connexion à Cassandra :

bash
docker exec -it mon-cassandra cqlsh
sql
-- 1. Création du keyspace (base de données)
CREATE KEYSPACE IF NOT EXISTS resto_NY 
WITH REPLICATION = { 'class' : 'SimpleStrategy', 'replication_factor': 1};

-- 2. Utilisation du keyspace
USE resto_NY;

-- 3. Création de la table Restaurant
CREATE TABLE Restaurant (
    id INT PRIMARY KEY,
    Name VARCHAR,
    borough VARCHAR,
    BuildingNum VARCHAR,
    Street VARCHAR,
    ZipCode INT,
    Phone text,
    CuisineType VARCHAR
);

-- Index sur le type de cuisine
CREATE INDEX fk_Restaurant_cuisine ON Restaurant (CuisineType);

-- 4. Création de la table Inspection
CREATE TABLE Inspection (
    idRestaurant INT,
    InspectionDate date,
    ViolationCode VARCHAR,
    ViolationDescription VARCHAR,
    CriticalFlag VARCHAR,
    Score INT,
    GRADE VARCHAR,
    PRIMARY KEY (idRestaurant, InspectionDate)
);

-- Index sur la note (grade)
CREATE INDEX fk_Inspection_Restaurant ON Inspection (GRADE);
Vérification :

sql
DESC Restaurant;
DESC Inspection;

V. Requêtes CQL et Réponses
N°	Requête CQL	Résultat / Remarque
1	SELECT * FROM Restaurant;	Liste complète des restaurants ✅
2	SELECT name FROM Restaurant;	Uniquement les noms ✅
3	SELECT name, borough FROM Restaurant WHERE id = 41569764;	Une seule ligne retournée ✅
4	SELECT inspectiondate, grade FROM Inspection WHERE idrestaurant = 41569764;	Toutes les inspections de ce restaurant ✅
5	SELECT name FROM Restaurant WHERE cuisinetype = 'French';	Restaurants français ✅
6	SELECT name FROM Restaurant WHERE borough = 'BROOKLYN' ALLOW FILTERING;	Requête acceptée après ajout de ALLOW FILTERING, car borough n’est pas une clé primaire.
7	SELECT grade, score FROM Inspection WHERE idrestaurant = 41569764 AND score >= 10 ALLOW FILTERING;	Idem, nécessite ALLOW FILTERING (filtrage sur un champ non indexé).
8	SELECT grade FROM Inspection WHERE score > 30 ALLOW FILTERING;	✅ La condition IS NOT NULL n’est pas supportée en CQL et a été omise.
VI. Difficultés rencontrées et solutions
Problème	Solution
Virtualisation désactivée	Activation dans le BIOS (Lenovo ThinkPad T14 : F1 → Config → CPU)
Espace disque insuffisant (C: = 89 Mo libre)	Nettoyage : docker system prune -a -f, vidage de la corbeille, suppression des fichiers temporaires (3.77 Go libérés)
Docker ne démarre pas	Redémarrage du PC et arrêt forcé des processus Docker dans le Task Manager
Erreur ALLOW FILTERING	Cassandra bloque le filtrage sur des colonnes non indexées pour des raisons de performance. Solution : ajouter ALLOW FILTERING (compris que cette méthode est à éviter sur de très gros volumes).
IS NOT NULL non supporté	En CQL, cette condition n’existe pas. On filtre uniquement sur les valeurs présentes.
